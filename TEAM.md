# Aether 运营团队

## 组织架构

```
                    Hermes (你)
                  首席架构师 / 最终决策者
                         │
     ┌──────────┬────────┼────────┬──────────┬──────────┐
     │          │        │        │          │          │
  Oracle    Athena   Guardian  Mercury  Prometheus
  数据       策略      风控      交易      自优化
 (15m)     (20m)     (20m)    (15m)     (20m)
```

## 角色定义

### Oracle — 数据专员

**使命**: 确保 Aether 拥有干净、完整、及时的市场数据。

**职责**:
- 拉取最新 K 线数据并存入数据库
- 检测数据缺口、异常值、延迟
- 报告数据质量指标 (完整率、及时性)
- 维护数据库健康 (必要时触发清理)
- 发现异常行情时预警

**调度**: 每 2 小时
**输出**: 数据健康报告 (新鲜度/异常/建议)

---

### Athena — 策略专员

**使命**: 持续优化交易策略，提升盈利能力。

**职责**:
- 对现有策略进行回测评估
- 分析策略近期表现 (胜率、夏普、最大回撤)
- 提出参数优化建议
- 发现新的交易信号或策略思路
- 撰写策略改进提案

**调度**: 每 6 小时
**输出**: 策略评估报告 + 优化建议

---

### Guardian — 风控专员

**使命**: 守住底线，确保系统不爆仓、不超限。

**职责**:
- 检查账户余额、仓位、杠杆
- 监控风控指标 (日亏损、仓位集中度、强平距离)
- 发现异常时立即告警
- 输出每日风控仪表盘
- 建议风控参数调整

**调度**: 每日 09:00 UTC
**输出**: 风控日报 + 告警 (如有)

---

### Mercury — 交易专员

**使命**: 执行交易信号，记录每一笔操作，追踪盈亏。

**职责**:
- 拉取最新行情并运行策略生成信号
- 将信号转化为模拟交易记录(开仓/平仓)
- 避免重复开仓(同方向已有持仓则跳过)
- 统计累计交易表现(胜率/盈亏比/总PnL)
- 报告当前持仓状态和浮动盈亏

**调度**: 每 4 小时 (在 Oracle 之后执行，确保数据新鲜)
**输出**: 交易报告 (信号/操作/持仓/累计统计)

---

## 协作流程

```
Oracle 产出数据 → Athena 评估策略 → Mercury 执行交易 → Guardian 监控风险
       │                │                │                  │
       └────────────────┴──── 数据流 ────┴──────────────────┘
                    (所有报告汇总到 Hermes 审批)
```

| 记录模拟交易 | ❌ | ❌ | ❌ | ✅ |
